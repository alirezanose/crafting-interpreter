Konsep Dasar Interpreter dan Kode Anda

Tujuan: Membangun program yang dapat menjalankan bahasa pemrograman kita sendiri (misalnya, Lox).

Proses utama dibagi 2 tahap:

1.  Kompilasi (Compilation)
    *   Apa itu: Proses menerjemahkan kode sumber yang bisa dibaca manusia (`print 1 + 2;`) menjadi format perantara yang lebih sederhana.
    *   Output: Bytecode.

2.  Eksekusi (Execution)
    *   Apa itu: Proses menjalankan Bytecode yang telah dihasilkan.
    *   Dilakukan oleh: Virtual Machine (VM).

---

Detail Konsep Penting:

1.  Token
    *   Apa itu: Unit terkecil yang bermakna dari kode sumber. Hasil dari proses Scanning (membaca karakter demi karakter dan mengelompokkannya).
    *   Contoh: Dari `"1 + 2;"` menjadi `TOKEN_ANGKA (1)`, `TOKEN_TAMBAH`, `TOKEN_ANGKA (2)`, `TOKEN_TITIK_KOMA`.
    *   **PENTING:** `Chunk` TIDAK menampung token. Token adalah input untuk Compiler.

2.  Bytecode
    *   Apa itu: Serangkaian instruksi numerik sederhana yang dipahami oleh VM kita. Ini adalah "bahasa mesin" untuk VM kita.
    *   Setiap instruksi diwakili oleh sebuah angka unik.
    *   Contoh: `OP_CONSTANT` (memuat angka), `OP_ADD` (menjumlahkan), `OP_RETURN` (mengakhiri eksekusi).
    *   Ini adalah output dari Compiler.

3.  OpCode (Operation Code)
    *   Apa itu: Nama spesifik untuk setiap instruksi bytecode. Setiap OpCode memiliki nilai numerik yang unik (misalnya, `OP_RETURN` mungkin bernilai 0, `OP_CONSTANT` bernilai 1, dll.).
    *   `enum OpCode` mendefinisikan daftar semua instruksi yang dapat dipahami oleh VM.

4.  Chunk
    *   Apa itu: Sebuah struktur data (array dinamis dari byte) yang **menampung seluruh urutan Bytecode**.
    *   Ini adalah "wadah" fisik tempat instruksi-instruksi bytecode disimpan setelah dikompilasi, siap untuk diberikan kepada VM.

5.  Virtual Machine (VM)
    *   Apa itu: Sebuah program yang kita tulis yang membaca dan menjalankan Bytecode dari `Chunk`.
    *   Bertindak sebagai CPU imajiner yang dirancang khusus untuk Bytecode kita.

6.  Disassembler (debug.h, debug.c)
    *   Apa itu: Alat bantu debugging.
    *   Peran: Menerjemahkan Bytecode dari `Chunk` kembali ke format yang bisa dibaca manusia. Berguna untuk memeriksa apakah Compiler menghasilkan Bytecode yang benar.

7.  Offset
    *   Apa itu: Posisi numerik (indeks) dari sebuah instruksi di dalam array `code` di `Chunk`.
    *   Contoh: `0000 OP_RETURN` berarti `OP_RETURN` dimulai pada indeks 0 di dalam array bytecode.
    *   Berguna untuk melacak lokasi instruksi, baik untuk VM (sebagai Program Counter) maupun untuk debugging.

---

Hubungan Antar Kode Anda dan Konsep:

*   `chunk.h`, `chunk.c`: Mengimplementasikan struktur `Chunk` (wadah Bytecode) dan cara mengelolanya (menulis, membebaskan).
*   `memory.h`, `memory.c`: Mengelola alokasi memori dinamis untuk `Chunk`.
*   `debug.h`, `debug.c`: Mengimplementasikan Disassembler (alat untuk membaca dan mencetak Bytecode).
*   `main.c`: Saat ini bertindak sebagai tempat pengujian untuk membuat `Chunk` dan menggunakan Disassembler.

---

Mekanisme Kerja Virtual Machine (VM):

1.  VM TIDAK membandingkan satu bytecode dengan yang lain.
2.  Tugas utama VM adalah **mengeksekusi** setiap instruksi bytecode satu per satu secara berurutan.

Siklus Inti VM (Fetch-Decode-Execute):

*   **Instruction Pointer (IP):** VM memiliki "penunjuk" internal (`ip`) yang berfungsi seperti penunjuk jari di buku resep. `ip` ini adalah offset ke instruksi bytecode berikutnya yang akan dijalankan di dalam `Chunk`.

*   **1. Ambil (Fetch):**
    *   VM membaca (mengambil) byte instruksi dari `Chunk` pada posisi yang ditunjuk oleh `ip`. Byte ini adalah sebuah OpCode (misalnya, nilai numerik `OP_RETURN`).

*   **2. Pahami (Decode):**
    *   VM melihat nilai OpCode yang baru saja diambil. Menggunakan struktur seperti `switch` statement, VM mengidentifikasi OpCode tersebut (misalnya, mengenali bahwa angka `0` berarti `OP_RETURN`).

*   **3. Jalankan (Execute):**
    *   Berdasarkan identifikasi OpCode, VM menjalankan kode C yang sesuai untuk melakukan aksi instruksi tersebut. Contoh: jika `OP_RETURN`, VM akan menghentikan eksekusi.

*   **Ulangi:** Setelah satu instruksi dijalankan, `ip` dimajukan ke instruksi berikutnya, dan siklus ini berlanjut sampai semua instruksi di `Chunk` selesai atau sampai instruksi `OP_RETURN` ditemukan.

### Bab 2: Mesin Virtual - Jantung Pelaksana (Versi Revisi dengan Contoh Kode)

Sekarang kita memiliki `Chunk` yang berisi bytecode, kita perlu sesuatu untuk **menjalankannya**. Inilah tugas Mesin Virtual (VM). VM adalah program yang kita tulis yang bertindak sebagai "CPU" imajiner untuk bytecode kita.

#### A. Konsep Kunci: Tumpukan (Stack) - Meja Kerja VM

**Kenapa harus ada Stack?**
Bayangkan Anda menghitung ekspresi matematika seperti `(5 - 3) * 4`. Anda tidak bisa melakukannya sekaligus. Anda harus menyimpan hasil sementara (`5 - 3` adalah `2`) di suatu tempat. **Stack** adalah "tempat penyimpanan sementara" ini. Ia bekerja dengan prinsip **LIFO (Last-In, First-Out)**, seperti tumpukan piring.

**Bagaimana ini diimplementasikan?**
*   **Struktur Data:** Di `vm.h`, `struct VM` memiliki array untuk menyimpan data stack dan sebuah pointer untuk melacak puncaknya.
    ```c
    // Contoh di vm.h
    typedef struct {
      // ...
      Value stack[STACK_MAX]; // Array untuk menyimpan data.
      Value* stackTop;      // Pointer ke slot kosong berikutnya.
    } VM;
    ```
*   **Operasi Stack:** Di `vm.c`, kita punya fungsi untuk memanipulasi stack.
    ```c
    // Contoh di vm.c
    static void push(Value value) {
      *vm.stackTop = value; // Simpan nilai di slot kosong teratas.
      vm.stackTop++;      // Majukan pointer ke slot kosong berikutnya.
    }

    static Value pop() {
      vm.stackTop--;      // Mundurkan pointer untuk menunjuk ke nilai teratas.
      return *vm.stackTop; // Kembalikan nilai tersebut.
    }
    ```

#### B. `struct VM` - Otak Operasi

**Kenapa harus ada `struct VM`?**
Untuk menjalankan tugasnya, VM perlu melacak beberapa hal sekaligus. `struct VM` menggabungkan semua "status" ini ke dalam satu wadah.

**Bagaimana ini diimplementasikan?**
```c
// Contoh di vm.h
typedef struct {
  Chunk* chunk;    // "Buku resep" bytecode yang sedang dieksekusi.
  uint8_t* ip;     // "Penunjuk jari" ke instruksi berikutnya di dalam resep.
  Value stack[STACK_MAX]; // "Meja kerja" untuk nilai sementara.
  Value* stackTop;      // Penanda di mana harus meletakkan barang berikutnya di meja kerja.
} VM;
```

#### C. Siklus Eksekusi Inti - Fungsi `run()` (`vm.c`)

**Kenapa harus ada `run()`?**
Ini adalah mesin penggerak utama VM. Ia berisi loop tak terbatas yang melakukan siklus **Fetch-Decode-Execute** berulang kali.

**Bagaimana ini diimplementasikan?**
Fungsi `run()` di `vm.c` adalah jantungnya. Mari kita lihat bagian-bagian utamanya.

1.  **Loop `for(;;)` dan Macro:**
    Loop ini memastikan VM terus bekerja. Untuk membuat kode di dalamnya lebih bersih, kita menggunakan *macro*.
    ```c
    // Contoh di vm.c, di dalam run()
    #define READ_BYTE() (*vm.ip++)
    #define READ_CONSTANT() (vm.chunk->constants.values[READ_BYTE()])

    for (;;) {
      uint8_t instruction;
      switch (instruction = READ_BYTE()) {
        // ... semua kasus OpCode ada di sini ...
      }
    }
    ```
    *   `READ_BYTE()`: Mengambil byte di posisi `ip` saat ini, lalu memajukan `ip`. Ini adalah langkah **Fetch**.
    *   `READ_CONSTANT()`: Menggunakan `READ_BYTE()` untuk mendapatkan indeks, lalu mengambil nilai dari kolam konstanta.

2.  **Decode & Execute dengan `switch`:**
    Setiap `case` di dalam `switch` adalah implementasi dari satu OpCode.

    *   **`case OP_CONSTANT:`**
        *   **Tujuan:** Mengambil nilai dari kolam konstanta dan meletakkannya di puncak stack.
        ```c
        // Contoh di vm.c, di dalam switch
        case OP_CONSTANT: {
          Value constant = READ_CONSTANT(); // Ambil nilai dari kolam konstanta.
          push(constant);                  // Dorong ke atas stack.
          break;
        }
        ```

    *   **`case OP_ADD:`** (atau `BINARY_OP` macro)
        *   **Tujuan:** Mengambil dua nilai teratas dari stack, menjumlahkannya, dan menyimpan hasilnya kembali ke stack.
        ```c
        // Contoh di vm.c, di dalam switch (jika menggunakan macro)
        case OP_ADD: BINARY_OP(+); break;

        // Apa yang sebenarnya dilakukan oleh BINARY_OP(+):
        // {
        //   double b = pop();
        //   double a = pop();
        //   push(a + b);
        // }
        ```
        Ini menunjukkan dengan jelas penggunaan stack: ambil dua, operasikan, kembalikan satu.

    *   **`case OP_RETURN:`**
        *   **Tujuan:** Mengakhiri eksekusi dan menampilkan hasil akhir.
        ```c
        // Contoh di vm.c, di dalam switch
        case OP_RETURN: {
          printValue(pop()); // Ambil hasil akhir dari stack dan cetak.
          printf("\n");
          return INTERPRET_OK; // Keluar dari fungsi run() dan hentikan loop.
        }
        ```

#### D. Menyatukan Semuanya - `interpret()` dan `main.c`

*   **`interpret(Chunk* chunk)` (`vm.c`):** Ini adalah fungsi "pintu depan" untuk VM.
    ```c
    // Contoh di vm.c
    InterpretResult interpret(Chunk* chunk) {
      vm.chunk = chunk;
      vm.ip = vm.chunk->code; // Atur 'ip' untuk mulai dari awal bytecode.
      return run();           // Mulai siklus eksekusi!
    }
    ```

*   **`main.c`:** Sekarang `main.c` bertindak sebagai "host" yang menyiapkan dan menjalankan VM.
    ```c
    // Contoh di main.c
    int main(...) {
      initVM(); // Siapkan VM (reset stack).

      Chunk chunk;
      initChunk(&chunk);
      // ... (Tulis bytecode ke chunk, seperti di contoh OP_CONSTANT_LONG Anda)

      interpret(&chunk); // Berikan chunk ke VM dan jalankan.

      freeVM(); // Bersihkan VM.
      freeChunk(&chunk);
      return 0;
    }
    ```
Dengan ini, semoga hubungan antara konsep ("stack", "VM", "siklus eksekusi") dan kode C yang sebenarnya (`struct`, `for`, `switch`, `push`, `pop`) menjadi jauh lebih jelas.
